The Challenge

DiNePa identified a critical gap in enterprise procurement: existing tools could not evaluate complex, multi-variable bidding events in real time. Their clients were running tenders with up to 80 suppliers across 100+ line items simultaneously. Traditional systems required 48–72 hours of post-event processing, delaying decision-making during active negotiations. DiNePa required a purpose-built SaaS platform capable of real-time bid evaluation across weighted line items while maintaining strict multi-tenant isolation.

Project Scope

We engineered the full lifecycle of this multi-tenant system, from the core auction engine to long-term production scaling.

  • 4 Isolated Portals: Super Admin, Client (Buyer Org), Buyer, and Supplier portals.
  • Real-Time Auction Engine: Support for Reverse, Forward, and Hybrid bidding formats.
  • Weighted Evaluation: Real-time recalculation across 100+ line items with distinct evaluation criteria.
  • Bulk Data Pipeline: CSV upload system with multi-step validation and currency normalization.
  • Multi-Language Support: Full localization for English, German, Spanish, and French markets.
  • Security Architecture: Backend-enforced Role-Based Access Control (RBAC) across all tiers.

Our Engineering Approach

1.

Domain Discovery

Mapped complex procurement variables—price, lead time, quality, and Incoterms—to build a multi-factor evaluation engine.

2.

Multi-Tenant Data Isolation

Implemented tenant-level partitioning at the database level with middleware validation to ensure zero cross-tenant exposure.

3.

Real-Time Event Processing

Architected an event-driven pipeline using Node.js and WebSockets to deliver sub-500ms updates under high concurrency.

4.

Secure Role Partitioning

Enforced role isolation at the backend routing level, ensuring strict security boundaries between Buyers, Suppliers, and Admins.

5.

Production-Grade Scalability

Validated the system through high-load simulations to ensure stability during peak European procurement surges.

Challenges & Solutions

Data Exposure Risk

Solved via database-level partitioning and strict middleware validation before any business logic execution.

High Recalculation Latency

Introduced an asynchronous processing layer, reducing bid evaluation recalculation to under 500ms.

Bulk Import Failure

Engineered a multi-stage CSV validation pipeline with a pre-submission preview, reducing errors to below 0.3%.
